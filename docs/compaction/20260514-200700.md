## Key Decisions
- **Remove unscoped permission variants** (`relationships:create`, etc.) → replace with scoped-only (`create:own`, `create:all`)
- **Two-rule resolution model:** (1) Within-role: most specific wins; (2) Across-roles: union, broadest (`:all`) overrides narrower (`:own`)
- **New roles introduced:** `relationship-admin` (full CRUD on rels), `relationship-viewer` (read-only audit/discovery)
- **object-type-admin role scoped strictly to object-types + objects** → no relationship permissions
- **Assignment constraint enforced:** Prevent both `type:action:own` AND `type:action:all` on same role via POST/PATCH/PUT validation
- **Cross-permission info leakage accepted:** `relationships:read:all` exposing object IDs is acceptable; handled by deliberate permission/role assignment
